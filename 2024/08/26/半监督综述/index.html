特点：（解耦，过滤标签）

为了解决一阶段 SSOD的基本问题

  • 伪标签低效 -> (多视图伪标签细化 MPR)

加入了MPR，只对教师网络置信度高的结果拿去给学生网络学习，不采用置信度低的

解耦就是把教师得到的分类跟画框的结果分开算，得到的是不同的分数：

2.DSL

（Dense learning based semi-supervised object detection）

特点：（精确的伪标签过滤（AF），增强标签稳定性（Aggregated teacher）， 不确定性一致正则化）

因为这篇论文采用的是FCOS那边的，所以画框的预测它是预测的中心点 + 长高，损失一共三部分：

$L{u, l} = \frac{1}{N{pos}} \sum{i} L{cls} + L{reg}(U{i, h,w}) + L{center}(U{i, h,w})$

还是跟正常的目标检测一样，只计算的正样本的损失

Adaptive Filtering Strategy（AF）

首先作者 分析了对10%COCO数据集进行预测的结果：

  • $TP+$ 可以理解为预测的类别正确的，但是画的框与GT 的 IOU高于0.5
  • $TP-$ 可以理解为预测的类别正确的，但是画的框与GT的 IOU低于0.5
  • $BG$ 就是背景或者预测的类别错误

想表达的意思就是，如果按照某一个固定的阈值去过滤教师产生的标签的话，可能会损失很多有用的信息

所以用多个阈值进行将实例分隔为三部分（前景，背景，可忽略的区域）

3.Dense Teacher

(Dense Teacher: Dense Pseudo-Labels for Semi-supervised Object Detection)

主流的生成伪标签的问题有以下三个：

  • 两个目标检测的网络都要使用EMA，用一个阈值（threshold）去除冗余框
  • 教师用一个阈值去过滤得到的预测的框
  • 正负样本的分配

本文就提出后处理的过程（NMS）是没有必要的，用区域划分的策略来抑制噪声，并且增强对关键区域的关注

将为标签得到的特征图，像FCOS中的一样，将特征图Feature Map通过分类头输出logic，
