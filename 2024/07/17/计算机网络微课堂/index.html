第一章 概述

1.因特网服务提供者ISP（Internet Service Provider)

（三大运营商）

isp三层：

2.因特网的组成

边缘部分

连接在网上的主机组成，是用户直接使用的，用来进行通信

核心部分

大量网络和连接这些网络的路由器组成，为边缘部分提供服务

3.三种交换方式

电路交换

电话机连接交换线的方式，（我感觉就是之前打电话的过程）

建立连接 -> 通话 -> 释放连接，效率很低

分组交换

分组交换机就是路由器，消息称为报文，

报文拆分之后，分别加上头部，用于路由器进行存储转发，到了之后，接收方接收分组，然后去掉头部还原报文

4.计算机网络的定义与分类

定义没有统一，可以定义为一些互相连接的/自治的计算机的集合

分类：

交换技术分类（之前的）

使用者分类（公用网，专用网）

传输介质分类（有线网络，无线网络）

覆盖范围分类：

  • 广域网WAN（覆盖范围广，可以跨国）
  • 城域网MAN（覆盖一个城市，作为城市骨干网）
  • 局域网LAN（在较小的范围内，校园网是多个局域网）
  • 个域网PAN（连接个人电子设备，用无线技术连接）

拓扑结构分类：

  • 总线型网络
  • 星形网络
  • 环形网络（局域网）
  • 网状型网络（广域网）

5.性能指标

常用的性能指标（速率，带宽，吞吐量，时延，时延带宽积，往返时间，利用率，丢包率）

速率：

比特：数据量的单位

次方进位，B, KB, MB, GB, TB

速率：传送比特的速率 , bit/s(b/s, bps)

进位，kb/s， Mb/s，Gb/s，Tb/s

带宽

单位时间内从网络中的某一点到另一点所能通过的最高速率

单位跟速率一致

吞吐量

单位时间内通过某个网络的数据量

受网络的带宽或额定速率的限制

时延

发送时延，传播时延，处理时延

哪个时延占主导要看具体情况，看发送的量

时延带宽积

传播时延 * 带宽

往返时间

RTT是网上的信息双向交互一次所需要的时间

利用率

信道利用率，网络利用率（并不是越高越好，越高说明用的人多，排队的时间越多）

最好的利用率是50%

丢包率（分组丢失率）

传输过程中分组数量与总分组数量的比率

  • 接口丢包率
  • 结点丢包率
  • 路径丢包率
  • 网络丢包率

分组丢失的两个情况：

  • 传输过程中出现误码，被丢弃
  • 网络拥塞，到队列满的交换机时被丢弃

不拥堵时丢包率0，轻度时1% - 4% ，重度拥堵时 5% - 15%

6计算机网络体系结构

常见的计算机体系结构

OSI体系结构（七层网络结构）法律上的国际标准

TCP/IP体系结构（四层网络结构） 实际的国际标准

TCP/IP：

原理体系结构：（五层）

将网络接口层分为之前的网络层跟物理层

计算机网络分层的必要性

  • 物理层：解决用什么信号来传输比特的问题
  • 数据链路层：解决分组在一个网络（或一段链路）上传输的问题
  • 网络层：解决分组在多个网络上传输（路由）的问题
  • 运输层：解决进程之间基于网络的通信问题
  • 应用层：解决通过应用进程的交互来实现特定网络应用的问题

分层思想举例

以浏览器与服务器通信为例

应用层：http

运输层：http + TCP

网络层： http + TCP + IP（ip数据报）

数据链路层： ETH + http + TCP + ETH（帧）

物理层：(ETH + http + TCP + ETH)转化为比特流，加上前导流，并且转化为电信号进行传输

计算机网络体系结构中的专用术语

实体：任何可以发送或接收信息的硬件或软件进程

对等实体：收发双方相同层次中的实体

应用层 — 应用层

协议：控制两个对等实体进行逻辑通信的规则的集合（http，tcp，ip等）

注意：只是逻辑通信，但是对等实体之间并不是直接存在通信

第二章 物理层

1.物理层的基本概念

在链路上传输比特0和1的问题
数据链路层只需要考虑使用物理层的服务

2.传输媒体

导引型传输媒体：

  • 同轴电缆
  • 双绞线
    两根线绞合在一起，（就是常见的网线，八根：蓝白，橙白，绿白，棕白）
  • 光纤，光纤很脆弱，所以会加上外面的结构组成光缆（通信容量大，传输损耗小等优点，25000 - 30000GHz的带宽）
  • 电力线

非导引型传输媒体（无线传输）（频率在的信号波

  • 无线电波（会大气层反射）
  • 微波（可以穿透大气层，中继站接力传播或用卫星做中继器）
  • 红外线（遥控器，点对点无线传输）
  • 可见光（实验阶段）

3.传输方式

串行传输/并行传输 同步传输/异步传输

4.编码与调制

编码：用于在数字信道上进行传输

调制：用于在模拟信道上进行传输

编码

不归零编码：码元不为0，高电平，低电平，存在同步问题，需要同步时间

归零编码：每次都回到0，不需要同步时间，用0区分每次

曼彻斯特编码：高跳低为1，低跳高为0，跳表示时钟

差分曼彻斯特编码：跳表示时钟，开始的时候的电平是否变化表示数据

第三章 数据链路层

1链路层概述

链路：从一个结点到相邻结点的一段物理线路，中间没有其他节点

数据链路：链路 + 实现通信协议的硬件和软件

数据链路层以帧为单位传输和处理数据

三个问题：

  • 封装成帧：加上帧头帧尾
  • 差错检测：有一个误差码，用于检验是否出错
  • 可靠传输

2.封装成帧

帧头和帧尾包含控制信息，作用之一是帧定界

透明传输是指数据链路层对上层交付的传输数据没有任何限制：

  • 面向字节的物理链路使用字节填充（就是在中间信息中与帧头帧尾重复的部分，前面加上一个转义字符一样的）
  • 面向比特的物理链路使用比特填充（相当于在中间的信息部分，每隔多少位就添加0，然后接收到的时候去掉这部分的0就能得到信息了）

3.差错检测

误码率：传输错误的比特占所有传输的比特的总数的比率

使用差错检测码

奇偶校验

发送的数据后面添加1为奇偶校验位，使整个数据中1的个数位奇数（奇校验）或偶数（偶校验）

如果奇偶性变化才会检查出误码，漏检率比较高

循环冗余校验CRC

  • 约定好一个生成多项式
  • 发送方基于待发送的数据和生成多项式计算出差错检测码（冗余码）
  • 接收方也将受到的数据用生成多项式进行计算，得到一个接收的冗余码，与发送过来的冗余码进行对比

检错码只能检测出帧在传输中是否出现了差错，不能定位错误

4.可靠传输

基本概念

  • 不可靠传输服务：直接丢弃有误码的帧
  • 可靠传输服务：想办法实现发送端发送什么，接收端就收到什么
  • 并不只存在于数据链路层，其他各层都可以实现可靠传输
  • 实现比较复杂，根据具体需求来选择

实现机制

停止-等待协议SW

  • 如果接收到的数据分组没有问题，接收方给发送方发送一个确认分组，然后发送方开始发送下一个数据分组
  • 如果收到的数据分组检测出差错，会给发送方法一个否认分组，然后发送方重新发送数据分组
  • 这种形式的发送方缓存中的信息不能发送之后就丢弃，需要收到接收方的确认信息之后才行
  • 可以加入超时重传，用于解决一整个数据分组丢失的情况，因为丢失之后接收方不会发送确认或者否认

回退N帧协议GBN

发送发有一个窗口（假设大小为），发送窗口内的数据，接收方一个一个的接收：

  • 如果整个窗口内的都没问题，就发送一个数，表示接收了多少个
  • 如果窗口内的数据有问题，接收了多少个就发送数字为多少，剩余的直接丢弃，然后发送方的窗口滑动之后，还是发送一整个窗口内的信息
  • 的大小为1，得到的就是停止-等待协议

选择重传协议SR

跟回退N帧协议一样，但是区别是，收到有误的之后，继续接收，只丢弃有误的，然后返回确认信息的时候也是，只返回确认无误的编号，然后发送窗口往前走，

5.点对点协议PPP

ppp协议是用于广域网（wan）中点对点连接，可以理解为像电话线中传输数据

mac帧是作用于局域网（lan）中，定义了如何在同一局域网的设备之间传输数据

Point-to-Point Protocol是目前使用最广泛的点对点数据链路层协议

帧格式：

FCS就是用于检验的，之前的CRC

实现透明传输的方法：（就是数据中有与收尾部重复的情况）

面向字节的异步链路：字节填充法，插入转义字符

面向比特的同步链路：比特填充法，插入比特0

6.共享式以太网

网络适配器（网卡）

内部与CPU之间的通信通过主板上的IO总线并行传输

外部与以太网之间的通信通过传输媒体（同轴电缆，光纤等）串行传输

实现物理层和数据链路层的功能

MAC地址（媒体接入控制Medium Access Control）

多个主机在同一个广播信道上，实现主机之间的通信，需要有一个唯一的标识，即一个数据链路层地址

MAC地址被称为硬件地址， 物理地址

以太网卡 / wifi网卡 都只有一个mac地址，交换机/路由器有多个mac地址，mac地址全球唯一

一个接口一个mac地址

  • 单播地址是一对一的，消息只发送给目标设备
  • 多播地址是一对多的，

网卡接受帧的情况：

  • mac地址是广播地址：（FF-FF-FF-FF-FF-FF）
  • mac地址与网卡上固化的mac地址相同，接受该帧
  • mac地址是网卡支持的多播地址，接受该帧

CSMA/CD协议

用于解决各站点争抢总线的问题，叫做载波监听多址接入/碰撞协议（在刚开始的共享式以太网，集线器把所有的设备连在一起）

在物理层扩展以太网

使用转发器，在物理层扩展以太网：（光纤调节器加集线器）

在数据链路层扩展以太网

使用网桥在数据链路层扩展以太网

网桥中有各个接口的mac地址表，收到帧之后，查询该表，

网桥收到帧之后进行登记（自学习），收到帧之后：

  • 如果表中有对应mac地址的信息，就转发相应的端口
  • 如果没有，就随意转发

透明网桥生成树协议STP：用于解决环路中死循环的问题。

7.交换式以太网

交换式集线器，实际上是有多个接口的网桥，称为以太网交换机或二层交换机

使用集线器连接的网络就称为共享式以太网，使用交换机连接的网络称为交换式以太网

交换机工作在数据链路层（包括物理层）

  • 当交换机的接口与计算机或交换机连接时，以全双工方式工作，在自身内部同时连通多对接口，不需要CSMA/CD协议
  • 当交换机连接的是集线器，该接口只能使用CAMA/CD协议

直通交换可以不缓存帧，直接进行转发，这样时延非常小，但是不检查差错

集线器扩展：扩大了广播域，扩大了碰撞域

交换机扩展：扩大了广播域，隔离了碰撞域

以太网没有数据重传机制

8.虚拟局域网VLAN

交换式局域网会增大广播域，巨大的广播域会导致一系列问题（广播风暴，难以管理和维护）

分割广播域的方法：

  • 使用路由器：路由器作用在网络层
  • 虚拟局域网

虚拟局域网概述

将局域网内的站点划分成物理位置无关的技术

同一vlan之间的站点可以直接进行通信，不同的vlan之间不能直接进行通信

虚拟局域网不是一种新型的网络，只是局域网提供给用户的一种服务

实现机制

常见的是基于以太网交换机的接口来实现vlan：

  • 交换机能够处理vlan标记的帧，IEEE 802.1Q帧
  • 交换机的各接口可以支持不同的接口类型

IEEE 802.1Q帧在mac帧的源地址跟类型之间，加了4字节的vlan标签

vlan标签组成：

主要是最后的VID，可以用于限制在同一vlan中转发

主机一般不处理802.1Q帧，是由交换机来处理，简称打标签

交换机转发802.1Q帧之后，可能会删除4字节的vlan标签，就变成了普通的mac帧，简称去标签

交换机的接口类型

分为Access和Trunk两种，如果之前没有进行vlan设置，默认是Access， 默认属于vlan1

交换机进行转发的过程，就是广播先通过交换机端口，打上端口对应的PVID标签，然后再转发的时候，将802.1Q帧转发到与其有相同的PVID值的端口上，去标签

Trunk与Access的区别：Trunk接口可以在一个物理链路上传输多个Vlan的数据，而Access接口只能承载一个vlan的数据流量

Trunk一般用于不同交换机之间，交换机与路由器之间进行连接，两个Trunk接口的PVID值最好相同

Trunk接口如果接受到的802.1Q帧的VID与接口的PViD相同，会去标签之后，当mac帧转发

如果不相同的情况，如上图的vlan2之间进行通信，会直接转发802.1Q帧

9.无限局域网

802.11 无限局域网（Wireless Local Area Network, WLAN）,人们喜欢称为Wi-Fi

  • 有固定基础设施的：预先建立的，多个固定的通信基站
  • 无固定基础设施的

第四章 网络层

网络层的主要任务就是将分组从源主机经过多个网络和多段链路传输到目的主机，主要任务可以划分为分组转发和路由选择

网络层向上提供两种服务 面向连接的虚电路服务
